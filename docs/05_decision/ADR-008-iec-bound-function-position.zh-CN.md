# ADR-008 — 把家庭功能位置绑定到 IEC 81346，并拒绝通用 `fulfills`

- 状态：提议
- 日期：2026-07-19
- 规范语言：英文
- 英文规范文本：[`ADR-008-iec-bound-function-position.md`](ADR-008-iec-bound-function-position.md)
- 映射证据：[`IEC 81346 边界审计 v0.1`](../../spec/mappings/iec81346/v0.1/README.zh-CN.md)

## 背景

HWM 夹具需要一个长期身份来表示“卧室阅读灯”，同时允许更换灯具、网络端点或 Agent。早期模型把这个身份称为 `Functional Slot`，并用 `fulfills` 把资产连到它。

这套用词有两个风险。第一，IEC 81346 已经覆盖跨生命周期与实现阶段的系统对象结构和参考代号。第二，`fulfills` 可能被理解为需求满足、候选适合、安装实现或身份同一，而 HWM 必须把这些 Claim 分开。

IEC 81346-1:2022 的官方公开摘要说明：参考代号标识系统对象，并在对象已实现时关联其对应部件；2022 版还增加了信息模型与对象关系标识方法。ISO 81346-12:2018 把该系列用于建筑工程与建筑服务，同时明确排除库存号、序列号和产品类型。这些公开材料足以确定概念所有权边界，但不足以确定精确的条款级关系编码。

## 决定

1. HWM 不把 `FunctionalSlot` 发布为 Core 本体类。
2. 生命周期 Profile 可以暴露**功能位置角色**：一个家庭稳定的 EntityRef，绑定到声明的 IEC 81346 参考代号系统及其功能导向结构。`Functional Slot` 只保留为迁移标签。
3. 绑定必须标识标准与版本、参考代号系统或上下文、参考代号、家庭 EntityRef 和适用生命周期区间。只有在合法取得规范条款后，才能固定精确的方面与关系编码。
4. 功能位置身份、具体制造资产身份、Product Model 与 Digital Endpoint 身份必须分开。IEC 参考代号不得充当库存号、序列号或产品型号标识。
5. HWM 拒绝把通用 `fulfills` 作为目标 Core 谓词。现有 `hwm-lifecycle:fulfills` 数据只作为兼容投影。
6. 实现至少必须区分：
   - 有时间边界的**资产—功能实现绑定**；
   - 需求或目标求值；
   - 规划上下文中的 Product Model 考虑；
   - 安装或空间放置；
   - 严格同一、替代表示与替换。
7. 实现绑定不能证明需求已满足、候选兼容、资产所有权、调试完成或端点可达。反过来，需求求值失败也不能推出资产当前没有实现该功能位置。
8. 当 Profile 和证据明确作用域与有效时间时，一个功能位置可以由多个资产共同实现，一个资产也可以参与多个功能位置。
9. `candidateFor` 只保留为带作用域的规划兼容投影。产品考虑应位于 `prov:Plan` 或带作用域的 `schema:ChooseAction` 中，不成为全局二元关系。
10. 精确的 IEC 81346 关系标识、方面元数据和序列化仍等待许可条款审查。在此之前，Profile 必须把绑定标为临时，且不得宣称符合 IEC。

## 后果

- 生命周期区别继续保留，但 HWM 不再与 IEC 81346 竞争领域词汇。
- 更换灯具或端点不会替换家庭功能位置身份。
- 不能再从资产安装意外推出 Goal Evaluation。
- 商品搜索、规划、安装、调试和运行继续属于不同认知阶段。
- 现有夹具仍可通过兼容别名读取，同时目标语义绑定可以逐步迁移。
- 剩余不确定性被缩小为可审计问题：IEC 精确线格式，而不是概念是否属于 HWM。

## 接受前仍需验证

1. 合法取得 IEC 81346-1:2022 与 ISO 81346-12:2018，并记录功能导向结构、对象关系和参考代号集的精确条款编号。
2. 请 IEC 81346 实践者挑战映射审计中的八个边界用例。
3. 增加一份带真实参考代号、且不使用 `hwm:FunctionalSlot` 或 `hwm:fulfills` 的外部语义投影。
4. 由独立实现证明“两个资产共同实现一个功能”和“一个资产参与两个功能”的用例。
