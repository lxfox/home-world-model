## 目的

本 Profile 限定多个设备、外界来源、空间路径和控制器共同作用于家庭环境时的预测，禁止 Agent 默认把单设备效果线性相加。

联合场景绑定参与资产／端点或商品选项、动作轨迹、控制模式、时间、空间、初始状态、外界 baseline、边界条件、共享资源和已知交互。日光、天气、人体散热、门窗、烹饪排放和相邻空间不能被伪装成设备贡献。

组合方式必须是 `direct_joint_model`、有验证域的 `validated_superposition`、带残差的交互模型、保守 envelope、保留分歧的 scenario ensemble，或明确 `not_composable`。单位一致只是必要条件，不表示物理可组合；两两独立不证明三设备以上独立；反馈控制器不能被静默改写为独立固定指令。

Assessment 检查 baseline closure、组件资格、空间／时间对齐、交互覆盖、非线性与饱和、共享资源、控制器、协方差、不确定性和联合验证。未知交互不是零交互；单体模型准确也不代表组合准确。

房间平均值不证明阅读面最低照度，平均温度不证明 occupied zone 舒适，总换气量不证明污染源附近空气质量。联合预测不能反向分摊为设备因果贡献，也不表示目标满足、选择、购买、合规、安全或行动权限。
