# ADR-013——把影响闭包限制在已声明通道内

- 状态：提议
- 日期：2026-07-19
- 规范文本：英文
- 英文规范文本：[`ADR-013-bound-impact-closure-to-declared-channels.md`](ADR-013-bound-impact-closure-to-declared-channels.md)
- 相关 Profile：[`受限影响闭包 Profile v0.1`](../../spec/profiles/bounded-impact-closure/v0.1/README.zh-CN.md)

## 背景

家庭在协调或授权 Action Proposal 前，需要知道什么可能受到影响。但单帧摄像、家庭成员名单、房间归属或一次全图遍历都不能回答这个问题。某人即使不在现场，也可能通过温控依赖、隐私处理、费用或控制利益受到影响；访客即使在房间内，也可能不在实质影响路径上。宠物、未来入住角色、共享资源和隐私数据主体也可能相关，却不能亲自回答确认请求。

因此，“完整受影响主体集合”只有在边界被明确声明时才安全。任何实现都无法证明自己发现了所有物理、社会、法律、经济、隐私或未来影响。

## 决策

1. HWM 区分描述性的**受到影响**与规范性的**参与资格**。影响报告可以指出某实体或不透明句柄处于影响路径上；只有 Authority policy 可以把它映射为确认、拒绝、通知、代表、安全或其他 Duty。
2. 影响闭包评估绑定一个不可变的 Action Proposal 标识与 revision、用途、声明影响领域与空间、决策时间、Authority Epoch 和提案时间范围。
3. 具名覆盖策略必须唯一匹配一条规则。规则声明必需影响通道、各通道认可的过程、报告新鲜度和必须覆盖的时间范围。
4. 影响通道是 Profile 局部策略标识，不是 HWM 的通用分类法。部署可使用物理暴露、隐私数据、共享资源、控制利益、费用分配或安全依赖等通道，但标签本身不授予权利。
5. 通道报告由声明的外部过程产生。HWM 验证其绑定与覆盖，不规定生成报告所用的仿真、感知、图算法、法律分析或模型。
6. `complete_for_declared_channels` 只表示每个必需通道都有一份当前、过程符合且覆盖声明时间范围的报告。不得把它语义缩写为“完整”“全局无影响”或“已知所有受影响实体”。
7. 必需报告缺失、过期、不可取得、不完整、绑定其他 Proposal revision、使用错误过程或无法覆盖时间范围时，覆盖结果为 `partial`，不得解释为空影响。
8. 没有或有多条适用规则、Authority 状态过期或不可取得、同一必需通道存在多份当前报告、或同一影响标识被冲突复用时，覆盖结果为 `indeterminate`。
9. 完整通道报告可以包含零个影响条目。这只表示该过程没有在该通道与时间范围报告实质或可能影响，不表示其他通道或范围外无人受影响。
10. 在场、家庭成员身份、房间归属和设备所有权可以作为声明过程的输入，但都不能单独成为受到影响或获得参与资格的必要或充分条件。
11. 影响条目保留所属通道和依据。即便不同通道引用同一实体，也不得据此折叠条目。
12. 未获准披露身份时，报告可以使用不透明参与者句柄。`access_denied`、脱敏或不允许披露数量，不得变成“没有受影响主体”。
13. Action Proposal revision 变化后，旧闭包不能继续用于行动。旧报告保留为历史 Record，但必须为新 revision 重算或显式重新生成。
14. 宠物、未来入住角色、资源和不透明主体可以出现在影响报告中，但这不推出法律行为能力、确认能力、家庭成员身份或特定监护人。
15. 评估是 Profile 类型化 Record；Core 不变。

## 组合关系

```text
精确 Action Proposal revision
  + 对特征与属性的预测 Effect Claims
  + 已声明影响通道策略与时间范围
  + 过程绑定的通道报告
  = 受限影响闭包评估

影响条目
  + Authority 影响程序映射
  = 异质程序性要求与不透明 slots

程序性要求
  + 回答、送达、复核与审计 Records
  = Procedure Fulfilment Assessment
```

## 结果语义

- `complete_for_declared_channels`：仅表示已满足声明的覆盖义务；
- `partial`：至少一个必需通道当前不能建立声明覆盖；
- `indeterminate`：策略、Authority 或报告完整性使评估无法唯一确定。

它们不是认知真值、行动授权、风险接受、法律同意或参与权决定。

## 标准边界

- SOSA/SSN 可描述 Actuation、Feature of Interest、Property、Procedure 与 Execution，但不定义家庭影响闭包。
- PROV-O 可描述报告与评估如何生成、使用、修订、受影响和归属；来源链不证明覆盖完整。
- BOT 与 Brick 可提供空间、包含、位置、设备、点位和系统拓扑；空间相邻不建立实质影响或权利。
- DPV 可描述个人数据处理、数据主体、影响评估、风险与权利。DPV 是 W3C Community Group Report，不是 W3C Recommendation，也不决定 HWM Authority policy。

HWM 保留的行为契约是：精确 Proposal 绑定、声明通道覆盖、失败关闭的不完整性、保护隐私的引用，以及与参与资格分离。

## 验收前验证

1. 独立实现复现 revision、过程、新鲜度、时间范围、缺失通道、重复报告、隐私和标识完整性用例。
2. 隐私评审挑战身份、在场、数量与不透明句柄披露。
3. 领域专家独立定义并测试至少物理影响和隐私影响过程。
4. Authority 评审证明影响条目不会自动变成投票权或同意权。
