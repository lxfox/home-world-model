## 目的

本 Profile 让家庭向 Agent 披露数据的过程可复现，但不虚构“开放世界中的绝对最小数据”证明。

规范链是：

`用途 + 精确消费／输出契约 -> 披露需求集 + 当前 Authority／披露 policy -> 经变换的披露包 -> 独立最小化评估 -> 有边界的释放／使用决定`

请求、必要、获准、已披露、充分、最小化和安全是不同判断。

## 相对最小化

请求者逐项声明语义操作、精度、空间／时间粒度、新鲜度、覆盖、来源、失败方式，以及保留、转发、训练、日志、删除和输出限制。“所有相关信息”“模型可能会用到”或方便性不构成必要性。

披露包逐项绑定来源和变换，可使用省略、聚合、泛化、化名、裁剪、特征提取、有限摘要、不透明 handle 或本地计算后只返回结果。评估检查用途充分性、逐项必要性、依赖闭包、精度比例、整包联结／推断风险、保留和输出泄漏。

`qualified` 只表示：在精确需求、policy、候选变换集合、当前时间和已知辅助信息下，没有已知的风险更低且仍满足契约的候选。它不是绝对、永久或全局最小证明。

## 组合边界

每个 work slot 独立绑定自己的需求、披露包和评估。父 Agent 的 View 不能直接传给工作者。多个单独合格的数据包合并后，必须重新评估整包联结风险。化名不等于匿名，聚合也不自动等于隐私安全。

输入合格不代表输出、日志、缓存、训练或下游工具调用合格。输出可能通过文字、坐标、embedding 或 trace 重构被隐藏内容，因此必须另有输出契约和检查。

撤销和到期只阻止未来使用，不改写历史；删除回执也不能证明受控边界之外的副本被删除。本 Profile 不授权行动，也不建立事实或证据适格性。
