# ADR-005 — Home World Model 核心定位与互操作 Profile

- 状态：提议（Proposed）
- 日期：2026-07-18
- 规范语言：英文
- 英文规范文本：[`ADR-005-home-world-model-core-positioning.md`](ADR-005-home-world-model-core-positioning.md)
- 相关规范：[`Home World Model 核心模型 v0.1`](../00_context/home-world-model-core-v0.1.zh-CN.md)

## 背景

项目最初的想法很宽：Agent 需要持久理解家庭空间、设备、人、时间、预期物理影响以及用户的纠错。研究与标准审查表明，Matter、Web of Things、SAREF、SOSA/SSN、BOT、Brick、IFC、NGSI-LD、PROV-O、ODRL、JSON-LD、SHACL 等体系已经提供了大量所需词汇。

因此项目必须决定：是创建一个大而全的新本体、依附某个运行时扩展，还是定义持久家庭知识与任意授权 Agent 之间尚缺失的互操作边界。

## 决策

1. HWM 是组合式互操作 Profile，不是新的通用本体、家庭自动化运行时、设备协议或 Agent 架构。
2. 通过映射或 Profile 复用现有标准。只有经过测试的情景证明存在真实语义缺口时，才引入 HWM 专有词汇。
3. 概念内核是 `H = (EntityRef, Claim, Record | Authority)`；Authority 是治理平面，而非普通家庭数据。
4. 可移植单位是 Household Knowledge Package。Evidence Resolver 将知识包和授权实时证据组合为受用途和时间约束的 World View。
5. 证据解析、Agent 规划、策略评估和执行是相互分离的角色与契约。
6. Core v0.1 定义四个交换契约：Household Manifest、Claim Envelope、World View 和 Action Trace，并提供一致性测试夹具。
7. World View 将可用性、认知状态、证据新鲜度、Claim 时间适用性与规范适用性保持为独立维度。有效时间区间统一为半开区间 [`from`, `to`)，精确时刻使用 `at`。
8. 行动结果是多维的。设备应答、物理观测、影响模型一致性、目标满足和用户接受不得折叠成单一成功值。
9. Core 要求简单 JSON 表达和扩展保留；允许 JSON-LD 映射，但不要求 RDF 引擎或图数据库。
10. 生物识别、人员追踪、活动推断、偏好协调算法、任意学习模型执行、完整 Authority 创世／恢复以及跨家庭信任推迟到 Core v0.1 之后。
11. 至少两个独立读取器／解析器通过相同一致性情景前，规范保持为讨论候选稿。

## 考虑过的替代方案

### 创建完整 HWM 本体

Core 阶段拒绝。它会重复成熟词汇、提高采用成本，并迫使语义共识依赖一个项目拥有所有家庭概念。

### 作为 Home Assistant 专用扩展

不作为规范边界。Home Assistant 是有价值的首个适配器和参考运行时，但家庭知识包必须能在替换该运行时后继续存在。

### 只扩展 Matter

不足以覆盖目标。Matter 已逐渐承载设备能力、上下文、偏好、解释和历史，但 HWM 还横跨设计意图、建筑空间、证据来源、持久生命周期身份、外部观测和独立于 Agent 的治理。

### 只发布叙事，不定义可测试契约

拒绝。叙事能吸引讨论，但不能建立互操作性，也不能暴露语义歧义。

## 影响

- 项目可以通过定义边界和测试保持精简，而不重做所有相邻标准。
- 实现可以采用关系、图、文档、文件、本地或云存储，只要可观察契约兼容。
- HWM 必须维护显式映射、Profile 版本和协商机制。
- 第一个可信里程碑不是消费产品，而是一个可移植参考知识包被两个独立实现兼容读取。
- 隐私敏感能力以后可以作为可选 Profile，而不污染最小 Core。

## 风险与缓解

- **Profile 歧义：**不同实现可能以不同方式解决冲突。通过定义输出契约和测试用基线解析器缓解，但不宣称一种算法普遍正确。
- **标准漂移：**复用标准会演化。Manifest 固定 Profile 版本并明确协商结果和不支持项。
- **词汇膨胀：**每个情景都可能提出新术语。新 Core 词汇必须先有缺口分析和一致性样例。
- **虚假确定性：**来源证明可能被误当成事实。分离 Claim、Record、证据关系和视图内解析状态。
- **离线撤权滞后：**被撤权 Agent 可能保留旧授权。使用 Authority 纪元和有界租约。

## 接受前所需验证

1. 两个独立开发的读取器往返处理同一个参考知识包，不丢失必需扩展或未知扩展数据。
2. 使用相同基线 Profile 和输入时，两者产生兼容的 World View 和行动评估。
3. 替换设备端点和 Agent 后，参考知识包仍保留绑定 IEC 的功能位置角色及相关历史。
4. 每个 HWM 专有 Core 术语都有针对现有标准的缺口说明。
